### 🎵 [HapticStudio](https://github.com/aIex-personal/bachelor-project) — `SvelteKit` `Web Audio API` `Electron` `PocketBase` `Docker`
Industry-sponsored full-stack platform developed with Hamsø Engineering ApS for designing, visualising, testing, and exporting digital haptic patterns — eliminating reliance on low-level haptic APIs and hardware-intensive workflows. Multi-track timeline editor, real-time waveform synthesis, multi-channel WAV export, RBAC, 135 unit tests + 61 E2E tests. Validated on real LRA actuator hardware. **Live at [studio.hamso.dk](https://studio.hamso.dk)**

### 👃 [SCENTINEL](https://github.com/aIex-personal/scentinel-machine-learning) — `Python` `FastAPI` `scikit-learn` `ESP32` `PostgreSQL` `Redis` `WebSockets`
Distributed intelligent odour detection system developed with ABENA A/S. Integrates ESP32-based e-nose sensors, FastAPI ML inference services, real-time WebSocket caregiver alerts, and separated operational/training databases. Two-stage ML pipeline processing H₂S, NH₃, VOC, and CH₄ sensor data. **Achieved 99.78% detection accuracy.**

### 🔍 [ML-Procurement](https://github.com/aIex-personal/ML-Procurement) — `Python` `scikit-learn` `pandas` `SQLite`
Machine learning platform for automated monitoring of EU public procurement data from the TED API. Modular pipeline covering ingestion, normalisation, feature engineering, anomaly detection (Isolation Forest, DBSCAN, K-Means), and reporting.

### 🏭 [Danfoss Heat Utility System](https://github.com/aIex-personal/Danfoss-Heat-Utility-System) — `C#` `Google OR-Tools` `GitHub Actions`
Optimises heat production scheduling across 4 district heating assets using linear programming. Minimises operating cost against live electricity spot prices and heat demand forecasts. Built in collaboration with Danfoss.

### ⚡ [ESP32-S3 Bare-Metal GPIO Driver](https://github.com/aIex-personal/esp32-s3-wroom-1_bare-metal-GPIO-driver) — `C` `PlatformIO` `CMake`
GPIO driver for the ESP32-S3-WROOM-1 in pure C — zero SDK, zero HAL. Direct memory-mapped register writes, atomic W1TS/W1TC set/clear, IO MUX pad configuration, stable-count software debouncer.
